% ── Abstract ──────────────────────────────────────────────────────────────────
\begin{center}{\large\bfseries Abstract}\end{center}
\noindent
We present a complete adaptive neurofeedback prototype addressing: \emph{given a
user's current physiological state and task context, what should the system do
next to improve the target brain signal?} Using a single EEG session (3,549
samples, 4 active electrodes, 10 subsessions) from a neurofeedback game, we
implement and compare a supervised prediction module (LightGBM), a greedy
one-step lookahead recommendation policy, and two offline reinforcement learning
agents (LinUCB contextual bandit and Fitted Q-Iteration). LightGBM reduces MAE
by 26.7\% over a persistence baseline (0.351 vs.\ 0.479) and achieves 67.6\%
\emph{directional accuracy} - the practically relevant metric for a system that
only needs to know whether improvement is expected. Both RL agents collapsed to
degenerate policies due to insufficient action coverage - a data-regime mismatch
rather than an algorithmic failure. We discuss evaluation metrics, the action
space problem, and a concrete path to a deployable real-time system.

\textbf{Contributions:}
\begin{enumerate}
  \item End-to-end EEG pipeline from raw signal to protocol recommendation in $<$100\,ms.
  \item Personalised z-score normalisation to a within-session baseline (subsession~0).
  \item Walk-forward cross-validation that strictly respects temporal ordering.
  \item Empirical demonstration that directional accuracy is the right metric where $R^2\approx 0$.
  \item Documented RL failure analysis establishing minimum data requirements.
\end{enumerate}

\section{Introduction}

Neurofeedback systems train users to self-regulate brain activity via real-time
EEG-derived feedback. The core challenge is \emph{adaptation}: a static protocol
ignores the highly individual and temporally non-stationary nature of EEG. An
adaptive system should adjust protocol parameters in response to the user's
current cognitive state - raising the threshold when the user performs well,
lowering it to prevent frustration, or switching modes on fatigue.

This framing is a sequential decision problem: choose the action (protocol
adjustment) that most likely improves the target neural signal. The appropriate
framework is the MDP, and the solution class is RL. However, RL requires data
with diverse action coverage, and a single neurofeedback session provides neither.

This work builds a minimal but complete prototype that navigates this tension
honestly: implementing the full RL pipeline while documenting precisely where and
why it fails, and providing a supervised prediction module that is immediately
deployable.

\section{Related Work}

\textbf{Adaptive neurofeedback.}
Traditional NFB protocols apply fixed thresholds throughout a session. Recent
work explores adaptive protocols that adjust difficulty in response to performance
\cite{navas2022, zander2011}. Ariel et al.\ \cite{ariel2021} provide a
comprehensive review of closed-loop adaptive NFB protocols. Key limitation:
existing adaptive systems rely on hand-designed heuristic rules, not learned
policies.

\textbf{ML for EEG.}
Lotte et al.\ \cite{lotte2018} review ML for EEG-based BCIs - the literature is
dominated by classification (motor imagery, P300) rather than regression of
continuous physiological signals. At 1-second resolution EEG is near-stochastic;
longer windows (10--30\,s) and CNNs substantially improve predictive power
\cite{schirrmeister2017}, motivating the rolling-window features used here.

\textbf{Offline RL in clinical settings.}
Gottesman et al.\ \cite{gottesman2019} show that offline RL on clinical data is
fundamentally limited by action coverage - directly applicable to our single-session
regime. Levine et al.\ \cite{levine2020} identify distributional shift between
logging and evaluation policies as the core challenge of offline RL.

\textbf{Contextual bandits for adaptive systems.}
LinUCB \cite{li2010} is the standard for personalised adaptive systems with
limited data. Soare et al.\ \cite{soare2020} apply contextual bandits to BCI
adaptation, demonstrating meaningful personalisation from tens of sessions.

\subsection{Non-RL Recommendation Policy}
\label{sec:nonrl}

A greedy policy selects the action predicted to yield the highest PV$(t+1)$:
\[
  a^*(t) = \arg\max_{a\in\{0,1,2\}}\; f_{\mathrm{LGB}}(\mathbf{x}(t) \mid \text{action} = a)
\]

% Fig 9
\begin{figure}[H]\centering
  \includegraphics[width=\linewidth]{../outputs/figures/nonrl_recommendations.png}
  \caption{Non-RL greedy policy recommendations overlaid on ProtocolValue
    trajectory. The policy collapses to 100\% Hold because the action feature
    has zero within-subsession variance (see \S\ref{sec:disc-failure}).}
  \label{fig:nonrl}
\end{figure}

\textbf{Limitation:} The \texttt{action} feature has zero within-subsession
variance (it is a per-subsession label), so LightGBM correctly assigns it zero
importance, producing identical predictions for all three action values and a
degenerate 100\% Hold policy.

\subsection{Reinforcement Learning Agents}
\label{sec:rl}

\textbf{MDP formulation} \cite{sutton2018}: State $s(t)$: 246-dim scaled vector.
Actions $A = \{0,1,2\}$. Reward $r(t) = \Delta\text{PV}$, clipped to
$[2^{\text{nd}}, 98^{\text{th}}]$ percentile. Discount $\gamma = 0.95$.

\textbf{LinUCB} \cite{li2010}:
\[
  \mathrm{UCB}_a(\mathbf{x}) = \theta_a^\top\mathbf{x}
    + \alpha\sqrt{\mathbf{x}^\top A_a^{-1}\mathbf{x}}, \quad \alpha=0.3
\]
Updated only on steps where the bandit's recommendation matches the logged action.

\textbf{FQI} \cite{ernst2005}: Iteratively fits $Q(s,a)$ using
GradientBoostingRegressor over 10 Bellman iterations ($\gamma = 0.95$).

\section{Discussion}

\subsection{What worked}

The supervised pipeline is clean, reproducible, and meaningfully better than
naive baselines. \textbf{67.6\% directional accuracy} allows threshold adjustments
to arrive just before improvement rather than after. The pipeline runs in $<$100\,ms
and is deployable on edge hardware.

\subsection{What did not work and why}
\label{sec:disc-failure}

Offline RL requires dense action coverage \cite{gottesman2019, levine2020}
- a finding established across healthcare and clinical domains. With
a single session and 70\% action~2 dominance:
\begin{itemize}
  \item \textbf{LinUCB:} $n=15$ matched steps - starved of signal for actions 0 and 1.
  \item \textbf{FQI:} Over-values action~0; Q-function never sees enough (state, action~0) pairs.
  \item \textbf{Non-RL:} Action feature has zero within-subsession variance; LightGBM assigns it zero importance.
\end{itemize}
This is a data regime mismatch, not a modelling failure.

\subsection{The action space problem}

No explicit real-time action column exists. A real deployment requires logging
each system action with a timestamp, enabling a properly labelled $(s,a,r,s')$
dataset. Without this, action-conditioned models conflate action effects with
natural state evolution - an identification problem unresolvable from
observational data alone.

\section{Conclusion}

We demonstrated a complete adaptive neurofeedback prototype - EDA, feature
engineering, supervised prediction, greedy policy, and two offline RL approaches
- in one week from a single EEG session. Key insights:

\begin{enumerate}
  \item \textbf{Directional accuracy, not $R^2$, is the right metric.}
  \item \textbf{Personalised baseline normalisation is essential} for cross-user generalisation.
  \item \textbf{Offline RL is data-hungry \cite{levine2020}.} Single-session skewed coverage is insufficient; the failure is informative.
  \item \textbf{The neurofeedback protocol works.} PV $+0.247$ over 9 subsessions ($R^2=0.94$).
\end{enumerate}

The primary bottleneck is data volume. The LinUCB architecture is ready for
real-time deployment; FQI is ready to scale with additional sessions and explicit
action logging.
